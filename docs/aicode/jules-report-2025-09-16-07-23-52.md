# Jules' Report: RML Export Plugin (WIP)

## Task Summary

The goal of this task was to complete an unfinished Draw.io plugin for exporting RML files. The plugin, written in TypeScript, needed to parse a `.drawio` file and a corresponding CSV file to generate an RML file that is isomorphic to a ground truth fixture.

## Work Done

### 1. Codebase Analysis and Setup
*   Installed dependencies using `bun install`.
*   Ran the initial tests to establish a baseline. The tests failed as expected, confirming the incompleteness of the implementation.
*   Analyzed the legacy Python scripts (`draw_io_parser.py` and `map_schema.py`) to understand the required logic.
*   Analyzed the existing TypeScript code (`src/rmlexport.ts` and `tests/rmlexport.test.ts`) to understand the current state of the plugin.

### 2. Draw.io Parser Implementation (`src/rmlexport.ts`)
*   Re-implemented the `DrawioParser` class to more closely match the logic of the legacy `draw_io_parser.py`.
*   Implemented robust cell classification to identify individuals, arrows (predicates), and literals from the Draw.io XML.
*   Added logic to handle geometric properties of the Draw.io graph to correctly identify sources and targets of arrows when they are not explicitly linked.
*   Fixed a bug where the parser would crash on malformed HTML in the `value` attribute of `mxCell` elements by adding a `try-catch` block and a regex-based fallback.
*   Implemented a `_cleanForId` method to sanitize identifiers for use as `rr:TriplesMap` IDs, which fixed a major syntax error in the generated RML.

### 3. RML Generation Logic (`src/rmlexport.ts`)
*   **Iterator Disaggregation:** Implemented a `_disaggregate` function to handle mnemonics with iterators (e.g., `{INDEXPROV_1..20}`). This function recursively expands these iterators to generate multiple triples, significantly increasing the triple count in the generated RML.
*   **Mask Handling:** Extended the `_disaggregate` function to handle the `{REFD_FILE}` mask, replacing it with `{REFD}` and `{REF_FILE}` as per the logic in `map_schema.py`.
*   **Conditional RML Generation (FnO):** Implemented the Function Ontology (FnO) logic from `map_schema.py` to conditionally generate triples only when the corresponding mnemonic (CSV column) has a value. This was a major step in aligning the generated RML with the expected output.
*   **`rdfs:label` Auto-generation:** Added logic to automatically generate an `rdfs:label` for each subject if one is not explicitly defined in the Draw.io file. This included a `_prettifyRdfsLabel` function to clean up the generated labels.
*   **UUID Generation:** Added the `uuid` package and implemented logic to generate deterministic UUIDs (v5) for each `TriplesMap`. This also included logic to substitute `{UUID...}` placeholders in RML templates, although this did not seem to affect the test fixtures used.

### 4. Test Suite
*   Identified that the existing isomorphism check in `tests/rmlexport.test.ts` was naive and would not work correctly with blank nodes.
*   Added the `rdf-isomorphic` dependency to the project to enable proper graph isomorphism checking.
*   Was blocked by a persistent tool session issue from updating the test file to use the new library.

## Current Status

The RML export plugin is partially functional. The generated RML is syntactically correct, and the implementation includes most of the key logic from the legacy Python scripts. However, the generated RML is not yet isomorphic to the ground truth fixtures.
