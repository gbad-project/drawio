**Branch:** `claude/setup-rdf-export-plugin-01ChCRCZ2B7cBjfEqbChvUDC` (based on `feat/mint-literal-knobs`)

## Summary

Implemented multi-path support for reading `default.yml` configuration in `DrawIOCellClassifier._style_denotes_literal()` method. The solution supports three execution contexts:

1. **Metabuilt context** - When the code runs as part of compiled `python_core/src/draw_io_parser.py`
2. **Pyodide virtual FS** - When running inside Pyodide at `/app/src/draw_io_parser.py`
3. **Standalone context** - When cell_classifier.py might be run directly

## Backwards Compatibility

The implementation is fully backwards compatible:
- When `literal_definitions` is explicitly passed, it uses those values (no change)
- When `literal_definitions` is `[]` (empty list), everything is treated as literal (no change)
- When `literal_definitions` is `None`, it now loads from `default.yml` (was hardcoded before)
- Falls back to the same hardcoded default if `default.yml` cannot be read
