```python
def _normalise_literal_definitions(value: Any) -> list[dict[str, str]]:
    """Normalize literal definitions to list of dicts with 'key' and 'value'."""
    if value is None:
        return []

    try:
        result = []
        for item in value:
            if isinstance(item, dict) and "key" in item and "value" in item:
                key = str(item["key"]).strip()
                val = str(item["value"]).strip()
                if key and val:
                    result.append({"key": key, "value": val})
        return result
    except (TypeError, KeyError):
        return []
```

#### B. Literal Detection Method (lines 721-739)
Rewrote `_style_denotes_literal()` to use configurable definitions:
```python
def _style_denotes_literal(self, cell: Element, style: str) -> bool:
    """Check if cell matches any literal definition."""
    # Check each literal definition
    for definition in self._literal_definitions:
        attr_name = definition.get("key", "")
        pattern = definition.get("value", "")
        if not attr_name or not pattern:
            continue

        # Get the attribute value from the cell
        attr_value = cell.attrib.get(attr_name, "")
        if not attr_value:
            continue

        # Check if the pattern exists in the attribute value
        if pattern in attr_value:
            return True

    return False
```

**Key Change:** Original method was static and hardcoded `"rounded=1" in style`. New method:
- Checks cell attributes dynamically based on literal_definitions
- Supports any attribute name (not just "style")
- Supports substring matching within attribute values

## Technical Decisions & Rationale

### 1. Field Name Mapping
**Decision:** Use different names in TypeScript vs Python
- TypeScript: `mintFromLiterals`, `literalDefinitions` (camelCase)
- Python: `mint_from_literals`, `literal_definitions` (snake_case)

**Rationale:**
- Follows language conventions
- TypeScript uses camelCase for object properties
- Python uses snake_case per PEP 8
- Mapping done in `buildParserConfigPayloadFromSettings()`

### 2. Literal Definitions Structure
**Decision:** Use array of `{key, value}` objects instead of single pattern

**Rationale:**
- Allows checking any cell attribute, not just "style"
- Enables multiple patterns (e.g., `style=rounded=1` OR `shape=ellipse`)
- More flexible than hardcoded check
- Follows pattern of metacharacter entries

### 3. SerialisationConfig Location
**Decision:** Override in `pre/internal/metadata` phase, not `core/rdf/data`

**Discovery Process:**
- Initially placed in `core/rdf/data` based on section comment in base file
- Tests failed with error: `internal_metadata_pre.SerialisationConfig.__init__() got unexpected keyword argument 'mint_from_literals'`
- Examined generated `draw_io_parser.py` to find actual location
- Found at line 2244 under section `# ===== pre.internal.metadata =====`
- Moved file and updated decorator to `@override(phase="pre", type="internal", role="metadata")`

**Rationale:**
- Must match exact phase/type/role of original class for meta-builder to apply override
- Original class is in pre-processing phase for internal metadata handling

### 4. Checkbox Defaults
**Decision:**
- `mintFromLiterals`: true
- `mintFromTypes`: false
- `mintFromArrows`: true

**Rationale:**
- `mintFromLiterals=true`: Preserves existing behavior, allows "literal-looking" strings to become URIs
- `mintFromTypes=false`: Types should typically be well-defined URIs, not minted from strings
- `mintFromArrows=true`: Preserves existing behavior, allows property names from arrows to be minted

### 5. Literal Definitions UI Pattern
**Decision:** Follow metacharacter entries pattern with add/remove buttons

**Rationale:**
- Consistent with existing UI
- Users already familiar with pattern
- Reuses existing helper functions (`createLabeledInput`, etc.)
- Allows unlimited entries

### 6. Method Signature Change
**Decision:** Changed `_style_denotes_literal()` from `@staticmethod` to instance method

**Rationale:**
- Needed access to `self._literal_definitions`
- Static method cannot access instance variables
- Breaking change, but internal method not part of public API
