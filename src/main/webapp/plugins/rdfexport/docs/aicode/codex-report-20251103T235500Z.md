# Template-aware typed individuals for RML

## Summary
- Reused the shared template detection logic in the RML/RDF serializer helper so DrawIO typed individuals accept `{template}` placeholders without tripping CURIE validation.
- Patched the RiC class verifier override to permit templates, added an override for metacharacter replacement to preserve absolute IRIs, and refreshed the generated parser.
- Updated the debug regression harness to skip template tokens when asserting rdf:type coverage, refreshed debug baselines, RML artifacts, and demo logs, and ensured the end-to-end workflow still xfails only on the previously documented cases.
